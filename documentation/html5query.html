Structure

Main structure

This template structure was generated using the generator-gulp-webapp

+---html5jquery
    +---app
    |   +---fonts
    |   +---images
    |   +---jade
    |   |   +---layouts
    |   |   +---partials
    |   +---scripts
    |   |   +---modules
    |   +---server
    |   +---styles
    |       +---app
    |       +---bootstrap
    +---bower_components
    +---dist
    +---node_modules
    +---test
    +---.tmp
 
Main folders
 
app
This folder contains the source files of the application that are used by gulp to generate the files to run the application.
 
dist
This folder is generated automatically from gulp with the source files ready to distribute
 
.tmp
This is folder is generated automatically from gulp when running the project in development mode
 
The task gulp clean removes the folders .tmp and dist so it's recommended to not add files manually to this folder if you are working with gulp

Class based layout
 
Other layouts can be activated with a classname  applied to the body
 
'layout-fixed' : Makes the sidebar and header fixed position

'layout-boxed' : Makes the content boxed with a limted width

'layout-dock' : Helps the layout dock (horizontal nav)

'layout-material': A layout similar to the used by the Material specification

'aside-offscreen' : Determines if the sidebar is visible on screen or not.

'footer-hidden': Allows to hide the footer

'aside-mini': Activates the collapsed sidebar with icons only. Text appears on hover.

'aside-right': Shows the sidebar right positioned

'in-app': When present, an application route is active. If not, we are in presence of a page.
 

Gulp

Since this template was built using the generator-gulp-webapp, most of the gulp tasks are the same listed in their documentation

Quickly get started

- In the root folder of this run the following command to install dependencies for nodejs

$ npm install

After install npm plugins, you can run:

$ gulp serve

to build a temporary folder and start a server with the files ready to work. Any change when this task is running will refresh the browser automatically.

When you are ready to deploy or distribute your application just run the following command to make a production-ready build of the app

$ gulp

Using HTML instead of JADE

The generator uses by default HTML but this template uses JADE as a default template engine. JADE is more simple and has many advantages over the classic HTML, that really improves the development process,

For those who want to use HTML instead I've included a gulp script that will work files in the app/html folder instead of jade. To use it you have two options

1. Set the gulp script in params

$ gulp --gulpfile gulpfile.html.js serve

Note that the last param is the task to run

2. Remove the original gulpfile.js and rename gulpfile.html.js to gulp uses it as default

 

Both version of the gulpfile works with LESS due to it supports pure css so it's not absolute necessary to learn how to use such preprocessor.

 

Compiling SCSS
 
 
 
To work with SCSS markup instead of LESS styles you have to use the files under the folder src/scss
 
Then, in order to compile this files run the following command
 
$ gulp --usescss
 

Themes

This themes uses class for background color to apply a theme.

You can see the available options in view Components -> Palette

Each .bg-* class has been created to consider style the elements within. You can see in the following markup example how classes are used to set the current theme.

<header class="bg-primary"></header>
<!-- Sidebar-->
<aside class="bg-white"></aside>
<!-- Main-->
<section>
   <!-- Content-->
   <div class="app"> ...
</section>
<!-- Page footer-->
<footer>...</footer>

 

To see and change the different configuration options, see the file
