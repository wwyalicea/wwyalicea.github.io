Enviroments

By using gulp you can work easily with two enviroments. One for development and another for production. Both of them uses the project root folder as the web root to serve the application and the most important difference is how the index file links assets. 

Development

This enviroment is what you get after uncompress the package. The index links each vendor assets necessary to start the application and each file of the javascript used in the theme (i.e. files located under the /src/js folder).
The css linked are bootstrap.css and styles.css files, where the second is the main stylesheet.

In this enviroment, the css is compiled from LESS using sourcemaps, allowing to see (debug) in Chrome DevTools from which LESS files come a certain rule.

Development enviroment is generated with gulp tasks 'default' and 'serve'

Production

This enviroment is what you should use to deploy your application. It concats all Javascript in two files, one for vendor assets and another for the theme scripts. 
Same for css, but only generates one single stylesheet (with bootstrap and the theme styles).

This enviroment is generated with gulp tasks 'build' and 'serve-build'

Class based layout
 
Other layouts can be activated with a classname  applied to the body
 
'layout-fixed' : Makes the sidebar and header fixed position

'layout-boxed' : Makes the content boxed with a limted width

'layout-dock' : Helps the layout dock (horizontal nav)

'layout-material': A layout similar to the used by the Material specification

'aside-offscreen' : Determines if the sidebar is visible on screen or not.

'footer-hidden': Allows to hide the footer

'aside-mini': Activates the collapsed sidebar with icons only. Text appears on hover.

'aside-right': Shows the sidebar right positioned

'in-app': When present, an application route is active. If not, we are in presence of a page.
 

Gulp

The structure of this theme is optimized to make use a GULP to build the production version.

To install gulp you will need first nodejs.

- Download NodeJS from https://nodejs.org/

Now you can install gulp and bower.
Open a console and run the following command

$ npm -g install bower gulp
Bower will require GIT to download new dependencies, so you may have to install it http://git-scm.com/book/en/v2/Getting-Started-Installing-Git

Now you have all this tools, you can move to the root folder of your project and run

$ npm install

This command will install node modules and vendor assets (if necessary) to allow build the theme for production and also use the live reload feature.

Main Gulp tasks supported

gulp
- this is default command, will compile the theme for development and will watch for changes

gulp build
- compiles the theme for production, will concat all assets and minify them in the app folder ready for deploy.

gulp clean
- this task will remove files generated by the compilation process

gulp compile
- just in case you want to compile for development without watch (optionally run clean task before)

gulp serve
- will start a server for development at http://localhost:3000

gulp serve-build
- will start a server with the production version of the theme

gulp plato
- this command will generate a plato report in the root folder of the theme https://github.com/es-analysis/plato

 

Most of the configurations (paths, names, ports, etc) can be changed by editing the file gulp.config.js

 

Compiling raw HTML

 

To work with HTML markup instead of JADE templates do the following

1. Create a folder to store the raw html files. Name it  /src/html

2. Copy the file index.html from the root folder into the html folder

3. Copy the folder /app/views into the html folder in step 1

4. Run the following command

$ gulp --usehtml

5. Done. If everything goes fine you can ignore jade templates and all changes in your html file will be replicated into the app folder

Optionally, you can drop the jade folder and run gulp clean to make sure that your tasks are generating the html files correctly.

 

Compiling SCSS

 

To work with SCSS markup instead of LESS styles you have to use the files under the folder src/scss

Then, in order to compile this files run the following command

$ gulp --usescss

 

 

 

 

 

RTL

RTL version is generated via the tool Gulp tasks runner.

It uses a plugin called rtlcss that converts the css from LTR to RTL inverting properties and taking other considerations.

You can learn more about from the documentation at https://github.com/MohammadYounes/rtlcss

Usage

Usage is simple. When you run gulp just add the following modifier 

$ gulp --usertl

The main styles of the template and Bootstrap will be generated in RTL  mode automatically.

 

 

 

Routes

This theme uses the UI Router provided by AngularUI team. This router allows to set states and and inherits dependecies for nested states.

To configure a route, see the module routes -> routes.config.js
For example: Here we are adding a route called myRoute.

        .state('app.myRoute', {
          url: '/myRoute',
          templateUrl: Route.base('myRoute.html'),
          resolve: {
            assets: Route.require( ... )
          }
        })
 
The resolve property is used to load assets required for a specific route. Those assets can be information requested to an internal api or a vendor assets required via lazy load.
 
Lazy load assets
 
This theme uses a constant to define only in one place the paths to each vendor file.
The constants is VENDOR_ASSETS and is defined in module routes -> vendor.constants.js
 
After adding the paths for a specific vendor plugin, is necessary to load in the route using the Route.require() function passing the name of the module or plugin to load.
 
For example, to require toaster for the above route, lets asume we haven't included yet in the theme, so we need to declare the new module in VENDOR_ASSETS constant
 
...
modules: [
 {name: 'toaster',           files: ['../vendor/angularjs-toaster/toaster.js',
                                     '../vendor/angularjs-toaster/toaster.css']},
...
 
After this, we will now to tell to the UIRouter to load this assets via the resolve property
 
        .state('app.myRoute', {
          url: '/myRoute',
          templateUrl: Route.base('myRoute.html'),
          resolve: {
            assets: Route.require( 'toaster¿' )
          }
        })
Note that the name if the module is the oficial module name.
You can use custom name for single scripts like jQuery plugins that doesn't init an Angularjs module.

Themes

This themes uses class for background color to apply a theme.

You can see the available options in view Components -> Palette

Each .bg-* class has been created to consider style the elements within. You can see in the following markup example (from file app.html) how the ng-class directive is used to set the current theme.

 

<!-- top navbar-->
<header ng-include="'templates/top-navbar.html'" ng-class="app.theme.topbar"></header>
<!-- Sidebar-->
<aside ng-include="'templates/sidebar.html'" ng-class="app.theme.sidebar"></aside>
<!-- Main-->

   <!-- Content-->
   <div ui-view="" autoscroll="false" ng-class="app.views.animation" class="app"></div>
</section>
<!-- Page footer-->

If you application doesn't use the theme changer you can simply remove all ng-class attributes and add the bg-* classname to each element.

Angular FullStack

A project boilerplate for an application based on the generator-angular-fullstack so you can find more details of this project at its website.

Angular fulltstack is Yeoman generator for creating MEAN stack applications, using MongoDB, Express, AngularJS, and Node - lets you quickly set up a project following best practices.

Main changed were made in the client folder. It has the following structure

+---client
    +---app
    |   +---account
    |   +---app
    |   +---dashboard
    |   +---_naut
    |       +---js
    |       +---less
    +---assets
    |   +---images
    +---bower_components
    +---components
    |   +---auth
    |   +---mongoose-error
    |   +---partials
    +---vendor
 
client/app/_naut: contains the main js and less files in similar structure like the original project for easy copy/paste required features.
 
client/app/bower_components: contains bower dependecies necessary to run the application (not for lazy loading)
 
client/app/vendor: contains bower dependecies use with lazy loading
 
clients/app/components/partials: contains partials files loaded from AngularJS ng-include (sidebar, header, etc)
 
 
Important: There are 2 bower.json files. One is located at the root level and the second one in the app folder. The second one is used only for lazy loading.
Since angular-fullstack doesn't support lazyloading I'had to found a workaround to download bower components in a place where the build system is not watching, in other case all components in the bower_components folder are included in the index and loaded with all views.
 

Visual Studio MVC5

This project is based on the Seed project and is a boilerplate tor an application using MVC5 and AngularJS.

The folder structure is

+---Naut
  +---App_Data
  +---App_Start
  +---Content
  +---Controllers
  +---Images
  |   +---user
  +---Langs
  +---Models
  +---Properties
  +---Scripts
  +---Vendor
  +---Views
      +---Dashboard
      +---Layout
      +---Shared
      +---Templates
Most of the folder are used like in any common MVC5 application. The following are the main folder used to implement the theme features:

Content: contains the css for the customized bootstrap stylesheet and the theme stylesheet

Scripts: contains the modular Javascript architecture used in the theme. 

Vendor: contains vendor assets. Some are linked from Bundles and other are requested using the ocLazyLoad module.

Views:

  /Dashboard: contains the dashboard demostration

  /Layout: contains the main markup for the theme layout

  /Shared: contains the main view loaded when the application starts

  /Templates: contains the templates requested from angular js ng-include directive (sidebar, header, etc)

 

All assets are placed in Bundles

For example

// Bootstrap Styles
bundles.Add(new StyleBundle("~/bundles/BootstrapStyles").Include(
          "~/Content/bootstrap.css"));
// Application Styles
bundles.Add(new StyleBundle("~/bundles/ApplicationStyles").Include(
          "~/Content/styles.css"));
// Application Script
bundles.Add(new ScriptBundle("~/bundles/ApplicationScripts")
    .Include("~/Scripts/app.module.js")
    .IncludeDirectory("~/Scripts/modules", "*.js", true)
);
//----------------------
// Vendor Bundle
//----------------------
bundles.Add(new ScriptBundle("~/bundles/VendorScripts").Include(
...
));

All bundles are defined in the file App-start/BundleConfig.cs
